# 백엔드 테스트

`backend` 디렉터리에서 실행합니다. 통합·E2E 테스트는 Docker가 필요합니다.

## 테스트 구조

`src/` 내부에 테스트 파일을 두지 않습니다. `test/` 디렉터리가 `src/` 구조를 미러링합니다.

| 디렉터리            | 유형        | 설명                                |
| ------------------- | ----------- | ----------------------------------- |
| `test/unit/`        | 단위 테스트 | DB 없이 도메인·순수 로직 검증       |
| `test/integration/` | 통합 테스트 | Docker DB 기반 Service 레이어 검증  |
| `test/e2e/`         | E2E 테스트  | Playwright 기반 API 엔드포인트 검증 |

### 레이어별 역할

| 분류        | 검증 범위                                              |
| ----------- | ------------------------------------------------------ |
| Unit        | 순수 로직, DTO 검증 규칙 (DB 없음)                     |
| Integration | Service 레이어 비즈니스 시나리오·엣지 케이스 (DB 필수)  |
| E2E         | Happy Path, JWT 가드 등 API 엔드포인트 문지기 역할     |

동일한 비즈니스 규칙을 여러 레이어에서 중복 검증하지 않습니다. Service 엣지 케이스는 Integration, API 인증·라우팅은 E2E에서 검증합니다.

### 코드 컨벤션

- `describe`·`it` 설명은 **한국어**로 작성
- 각 테스트는 `// given`, `// when`, `// then` 주석으로 구분

## 사전 준비: 테스트 환경 YAML

`backend/enviorment/test-enviorment.yml` 파일이 필요합니다.

```bash
# 프로젝트 루트에서 실행
mkdir -p backend/enviorment

cat > backend/enviorment/test-enviorment.yml << 'EOF'
database:
  url: postgresql://prisma:prismapass@localhost:5433/inflearn_clone-test?schema=public
jwt:
  authSecret: test-jwt-secret
server:
  port: 8000
aws:
  s3Region: ap-northeast-2
  mediaFileBucketName: test-media-bucket
  cloudFrontDomain: https://localhost
EOF
```

테스트 DB는 로컬 개발 DB(포트 `5432`)와 분리되어 포트 `5433`을 사용합니다.

## 단위 테스트

```bash
cd backend
pnpm test:unit
```

## 통합 테스트

```bash
cd backend
pnpm test:int-setup   # 테스트용 DB 컨테이너 구동 및 스키마 동기화 (최초 1회)
pnpm test:int
```

`test:int-setup`은 `NODE_ENV=test`로 `docker-compose.test.yml`을 기동하고 `prisma db push`를 실행합니다.

## E2E 테스트

Playwright가 테스트 실행 시 Docker DB 구동 → NestJS 서버 기동을 **자동으로** 수행합니다.

```bash
cd backend
pnpm test:e2e
```

Playwright UI 모드:

```bash
pnpm test:e2e-ui
```

인프라를 미리 준비해 두고 E2E만 반복 실행하려면:

```bash
pnpm test:e2e-setup
SKIP_TEST_INFRA_SETUP=1 pnpm test:e2e
```

## 전체 테스트

```bash
cd backend
pnpm test:int-setup   # 통합·E2E에 필요한 DB 인프라 (최초 1회)
pnpm test:all
```

`test:all`은 setup을 자동 실행하지 않습니다. 최초 실행 전 `test:int-setup`을 먼저 수행하세요.
