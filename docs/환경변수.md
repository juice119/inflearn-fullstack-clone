# 환경변수

## Backend — YAML 설정

경로: `backend/enviorment/{환경}-enviorment.yml`  
환경명은 `NODE_ENV` 값과 대응합니다 (`local`, `test`).

| 키 | 타입 | 설명 |
| --- | ---- | ---- |
| `database.url` | string | PostgreSQL 연결 URL |
| `jwt.authSecret` | string | JWT 서명 시크릿. Frontend `AUTH_SECRET`과 동일 값 사용 |
| `server.port` | number | API 서버 포트 (로컬 기본 `8000`) |
| `aws.s3Region` | string | S3 리전 (예: `ap-northeast-2`) |
| `aws.mediaFileBucketName` | string | 미디어 파일 S3 버킷명 |
| `aws.cloudFrontDomain` | string | CloudFront 도메인 URL |
| `aws.profile` | string (선택) | 로컬 AWS CLI 프로필명. 설정 시 해당 프로필 자격 증명 사용 |

### 로컬 예시 (`local-enviorment.yml`)

```yaml
database:
  url: postgresql://prisma:prismapass@localhost:5432/inflearn_clone?schema=public
jwt:
  authSecret: your-local-secret
server:
  port: 8000
aws:
  s3Region: ap-northeast-2
  mediaFileBucketName: your-bucket-name
  cloudFrontDomain: https://your-cloudfront-domain
  profile: your-aws-cli-profile
```

### 테스트 예시 (`test-enviorment.yml`)

```yaml
database:
  url: postgresql://prisma:prismapass@localhost:5433/inflearn_clone-test?schema=public
jwt:
  authSecret: test-jwt-secret
server:
  port: 8000
aws:
  s3Region: ap-northeast-2
  mediaFileBucketName: test-media-bucket
  cloudFrontDomain: https://localhost
```

## Frontend — `.env`

경로: `frontend/.env` (gitignore 대상)

| 변수 | 설명 |
| ---- | ---- |
| `DATABASE_URL` | PostgreSQL 연결 URL. Backend와 동일 DB 사용 |
| `AUTH_SECRET` | NextAuth JWT 서명 시크릿. Backend `jwt.authSecret`과 **동일** |
| `API_URL` | Backend API 베이스 URL (기본 `http://localhost:8000`) |

### 예시

```env
DATABASE_URL=postgresql://prisma:prismapass@localhost:5432/inflearn_clone?schema=public
AUTH_SECRET=your-local-secret
API_URL=http://localhost:8000
```
