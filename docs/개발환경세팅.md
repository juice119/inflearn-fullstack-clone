# 개발 환경 세팅

## 사전 요구사항

- [Node.js](https://nodejs.org/) 20 이상
- [pnpm](https://pnpm.io/)
- [Docker](https://www.docker.com/) / Docker Compose
- 미디어 업로드(프로필·커버·영상) 사용 시 [AWS CLI](https://aws.amazon.com/cli/) 및 S3/CloudFront 접근 권한

## 프로젝트 구조 안내

루트·`backend`·`frontend`가 각각 독립된 pnpm 패키지입니다. 루트 `pnpm install`은 Husky·lint-staged용이며, **앱 실행에는 `backend`와 `frontend` 각각의 `pnpm install`이 필수**입니다.

DB 스키마의 기준은 `backend/prisma/schema.prisma`입니다. `pnpm db:push`는 backend에서만 실행하면 되며, frontend Prisma는 NextAuth용 일부 모델만 포함합니다.

## 로컬 실행

### 1. 저장소 클론 및 의존성 설치

```bash
git clone https://github.com/juice119/inflearn-fullstack-clone.git
cd inflearn-fullstack-clone

# Husky pre-commit 훅 (선택이지만 권장)
pnpm install

# 백엔드·프론트엔드 의존성 (필수)
cd backend && pnpm install && cd ..
cd frontend && pnpm install && cd ..
```

### 2. PostgreSQL 실행 (로컬 개발용)

```bash
cd backend
docker compose up -d
```

| 항목     | 값               |
| -------- | ---------------- |
| Host     | `localhost`      |
| Port     | `5432`           |
| User     | `prisma`         |
| Password | `prismapass`     |
| Database | `inflearn_clone` |

> 테스트용 DB는 포트 `5433`을 사용합니다. 로컬 DB와 충돌하지 않도록 [백엔드테스트](./백엔드테스트.md) 문서를 참고하세요.

### 3. Backend 설정 및 실행

`backend/enviorment/local-enviorment.yml` 파일을 생성합니다. (디렉터리명 `enviorment`는 프로젝트 고유 철자입니다.)

```bash
# 프로젝트 루트에서 실행
mkdir -p backend/enviorment

cat > backend/enviorment/local-enviorment.yml << 'EOF'
database:
  url: postgresql://prisma:prismapass@localhost:5432/inflearn_clone?schema=public
jwt:
  authSecret: your-local-secret
server:
  port: 8000
aws:
  s3Region: ap-northeast-2
  mediaFileBucketName: your-bucket-name
  cloudFrontDomain: https://your-cloudfront-domain
  profile: your-aws-cli-profile
EOF
```

`aws.profile`은 로컬 AWS CLI 프로필명입니다. 미디어 업로드를 사용하지 않을 때도 YAML 검증을 통과하려면 `aws` 섹션 자체는 필요합니다. 상세 항목은 [환경변수](./환경변수.md)를 참고하세요.

```bash
cd backend
pnpm db:gen
pnpm db:push
pnpm seed          # 카테고리 시드 데이터 (선택)
pnpm start:local-dev
```

서버가 정상 기동되면 아래 주소에서 확인할 수 있습니다.

- API 서버: http://localhost:8000
- Swagger 문서: http://localhost:8000/docs

디버그 모드가 필요하면 `pnpm start:local-debug` 또는 VS Code **NestJS: pnpm Debug** 구성을 사용합니다.

### 4. Frontend 설정 및 실행

`frontend/.env` 파일을 생성합니다.

```env
DATABASE_URL=postgresql://prisma:prismapass@localhost:5432/inflearn_clone?schema=public
AUTH_SECRET=your-local-secret
API_URL=http://localhost:8000
```

> `AUTH_SECRET`은 backend `jwt.authSecret`과 **동일한 값**이어야 합니다. NextAuth가 발급한 JWT를 Backend API 인증에 사용합니다.

```bash
cd frontend
pnpm db:gen
pnpm dev
```

프론트엔드: http://localhost:3000

### 5. OpenAPI 클라이언트 생성

`frontend/generated/`는 gitignore 대상입니다. **최초 클론 후** Backend가 실행 중인 상태에서 한 번 생성해야 합니다.

```bash
cd frontend
pnpm openapi-ts
```

API 스펙이 변경되었을 때도 동일한 명령으로 재생성합니다.

## 트러블슈팅

| 증상 | 확인 사항 |
| ---- | --------- |
| `설정 파일 검증 실패` | `local-enviorment.yml`에 `aws` 섹션이 있는지 확인 |
| `local-enviorment.yml`을 찾을 수 없음 | `NODE_ENV=local`로 실행했는지 확인 (`pnpm start:local-dev` 사용) |
| Prisma DB 연결 실패 | `docker compose up -d` 후 컨테이너가 running 상태인지 확인 |
| 포트 충돌 (5432) | 기존 PostgreSQL 프로세스 종료 또는 `docker-compose.yml` 포트 변경 |
| OpenAPI 타입 import 오류 | Backend 기동 후 `pnpm openapi-ts` 실행 |
